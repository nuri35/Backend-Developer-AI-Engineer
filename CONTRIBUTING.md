# Contributing

Thank you for your interest in contributing to this roadmap.

This is a community resource. If you find something missing, wrong, or unclear — please open a PR or an issue. Every contribution makes this better for everyone.

---

## What kind of contributions are welcome?

### High value
- Fixing incorrect technical information
- Adding missing topics that are important for backend developers entering AI
- Adding Node.js/TypeScript code examples where they are missing
- Improving explanations that are unclear or incomplete
- Adding new resources that are genuinely useful (free courses, quality videos, official docs)
- Reporting broken links

### Also welcome
- Fixing typos and grammar
- Improving formatting
- Adding real-world project ideas
- Sharing your experience transitioning from backend to AI engineering

### Not in scope
- Python-only content (this roadmap focuses on Node.js/TypeScript)
- ML model training content (this roadmap is for integrating existing models)
- Promotional content or paid tool recommendations without genuine value

---

## How to contribute

### For small changes (typo, broken link, small correction)

1. Fork the repository
2. Make your change
3. Open a Pull Request with a short description of what you changed and why

### For larger changes (new topic, new section, significant rewrite)

1. Open an issue first to discuss the change
2. Wait for feedback before writing a lot of content
3. Fork and make the change
4. Open a Pull Request with a clear description

---

## Style guide

### Language
- Write in clear, simple English (B1–B2 level)
- Short sentences — prefer 15 words or fewer per sentence
- Avoid jargon when a simpler word works
- If you use a technical term for the first time, explain it

### Code examples
- Use TypeScript (not Python)
- Use NestJS patterns where applicable
- Keep examples focused — show one concept, not everything at once
- Include types and error handling in real examples

### Formatting
- Use the same structure as existing topic files
- Every topic file should have: What is this topic, Core concepts, Code examples, Key terms, What to do next
- Use tables for comparisons
- Use code blocks for all code

### Accuracy
- Only include information you have personally verified
- If you are not sure about something, say so or leave it out
- Include sources where possible

## Code of conduct

Be helpful and respectful. This is a learning resource — people at all levels are welcome.

If you see something wrong, point it out kindly. The goal is accuracy, not criticism.
